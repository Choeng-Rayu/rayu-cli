# Specialist Swarm Test Scenario

## Task
Build a simple Invoice Management MVP for a Cambodian market:
- Users can register/login (KHR/USD dual currency support, Khmer+English UI)
- Create and manage invoices (PDF export, basic reporting)
- Integration with Bakong payment (Cambodia's CBDC)
- Mobile app support for invoice viewing

## Requirements
- Stack: Choose tech (PA-AGENT decides)
- Database: Design schema with dual-currency support (DB-AGENT)
- Backend: REST API routes, auth middleware (BE-AGENT, implements SEC-AGENT's auth)
- Security: Auth design, RBAC, validation, Bakong API keys handling (SEC-AGENT)
- Frontend: Dashboard, invoice form, PDF viewer (FE-AGENT)
- Mobile: Flutter app with offline support (MOB-AGENT)
- DevOps: Docker, CI/CD for containerization (DO-AGENT)

## Test Goals
1. **PA-AGENT** writes tech stack decision (Node.js/Next.js, Prisma, PostgreSQL)
2. **DB-AGENT** designs invoice schema with dual-currency columns
3. **BE-AGENT** implements API routes using PA stack + DB schema
4. **SEC-AGENT** defines auth flow, validates against OWASP
5. **FE-AGENT** builds dashboard components using BE routes
6. **MOB-AGENT** plans Flutter architecture matching BE API
7. **DO-AGENT** creates Dockerfile + CI/CD
8. **Verify context sharing** — each specialist reads prior specialists' output
9. **Test permissions** — do tool calls bubble to user or get stuck?
10. **Test memory** — do specialists save learnings to MEMORY.md?
